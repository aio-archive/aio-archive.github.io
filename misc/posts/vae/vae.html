Probabilistic modeling is one of the most fundamental ways in which we reason about the world. This paper spearheaded the integration of deep learning with scalable probabilistic inference (amortized mean-field variational inference via a so-called reparameterization trick), giving rise to the Variational Autoencoder (VAE). The lasting value of this work is rooted in its elegance. The principles used to develop VAEs deepened our understanding of the interplay between deep learning and probabilistic modeling, and sparked the development of many subsequent interesting probabilistic models and encoding approaches.

Preliminaries

Before diving into VAE, we first explore about the (statistical) tools that are useful later to understand how VAE works.

Jensen’s inequality

Jensen’s inequality is a useful concept for any convex function that has many applications in generative models. A convex function can be defined as follows:

Definition of convex function (Boyd and Vandenberghe 2004)

A function \(f: \mathbb{R}^n \mapsto \mathbb{R}\) is said to be convex if its domain \(\textbf{dom}f\) is a convex set and if for all \(x,y \in \textbf{dom}f\), \(\theta \in [0,1]\), we have:

\[f(\theta x + (1-\theta) y) \leq \theta f(x) + (1-\theta) f(y)\]

Boyd, Stephen, and Lieven Vandenberghe. 2004. Convex Optimization. Cambridge University Press.

For example, consider the figure below where the figure on the left is the graph of a convex function \(f(x) = x^2\), the line (sometimes also referred to as chord) between any two points \(x,y\) on the graph lies above the graph, i.e. every chord is below the function. In contrast, the function on the right does not have such property so it is not a convex function. The definition that we have for a convex function is just a generalization and mathematical expression of this property.

Given a convex function \(f\) and a random variable \(X\), Jensen’s inequality states that:

Jensen’s inequality

\[\mathbb{E}[f(X)] \geq f(\mathbb{E}[X])\]

Later in this section, we will apply the Jensen’s inequality to derive the Evidence Lower Bound, which is the workhorse of VAE.

KL Divergence

Sometimes it is useful for us to have a measure \(D\) to quantify the difference between two distributions. We require that measure to have the following properties (Murphy 2023):

  • Continuous in its arguments
  • Nonnegativity
  • Permutation invariant
  • Monotonic
  • Satisfy the natural chain rule

KL divergence turns out to be that useful tool and it is defined as follows:

KL Divergence

\[D_{KL} = \int p(x) \log \dfrac{p(x)}{q(x)} dx\]

Interested readers can refer to (Murphy 2023) to see the proof that KL Divergence satisfies all of the desiderata.

Murphy, Kevin P. 2023. Probabilistic Machine Learning: Advanced Topics. MIT Press. http://probml.github.io/book2.

Autoencoder

While VAEs (Kingma and Welling 2013) and the deterministic autoencoder (AE) are very similar, there are two main differences:

Kingma, Diederik P., and M. Welling. 2013. “Auto-Encoding Variational Bayes.” In International Conference on Learning Representations. Vol. abs/1312.6114. 1. Explorandum Ltd.

  1. in the AE, the objective is the log likelihood of the reconstruction without any KL term

  2. The encoding is deterministic, so that the encoder network just needs to compute \(\mathbb{E}[\mathbf{z} \vert \mathbf{x}]\) and not \(\text{Var}[\mathbf{z} \vert \mathbf{x}]\).

The main advantage of VAE over a deterministic autoencoder is that it defines a proper generative model that can create natural-looking novel images by decoding prior samples \(\mathbf{z} \sim \mathcal{N}(0, \mathbf{I})\). This is in constrast with the autoencoder, where it only knows how to decode the latent codes derived from the training set, so it does poorly on random inputs, as shown in Figure

VAE Setting

The autoencoder has an input variable \(\mathbf{x}\) and a latent variable \(\mathbf{z}\), where \(\mathbf{x}\) is an input image and \(\mathbf{z}\) is some latent vector in high dimensional space.

The term variational in Variational Autoencoder refers to the fact that we use probability distributions to describe \(\mathbf{x}\) and \(\mathbf{z}\).

There are four main distributions that we need to consider for VAE

  • \(p(\mathbf{x})\) is the distribution of \(\mathbf{x}\), we never have access to this distribution and the goal of generative models in general is to find ways to draw samples from \(p(\mathbf{x})\).

  • \(p(\mathbf{z})\) is the distribution of the latent vector \(\mathbf{z}\). We assume that \(p(\mathbf{z}) \sim \mathcal{N}(0, \mathbf{I})\).

  • \(p(\mathbf{z} \vert \mathbf{x})\) is the conditional distribution that is associated with the encoder, which tells us the likelihood of \(\mathbf{z}\) given \(\mathbf{x}\).

  • \(p(\mathbf{x} \vert \mathbf{z})\) is the conditional distribution that is associated with the decoder that tells us the likelihood of \(\mathbf{x}\) given \(\mathbf{z}\).

Proxy distributions

Since we don’t have access to the two high-dimensional distributions associated with the encoder \(p(\mathbf{z} \vert \mathbf{x})\) and the decoder \(p(\mathbf{x} \vert \mathbf{z})\), we need to come up with a way to approximate it. Consider the following two distributions:

  • \(q_{\phi}(\mathbf{z} \vert \mathbf{x})\), which is the proxy for \(p(\mathbf{z} \vert \mathbf{x})\) and we assume this to be a Gaussian distribution.
  • \(p_{\theta}(\mathbf{x} \vert \mathbf{z})\), which is the proxy for \(p(\mathbf{x} \vert \mathbf{z})\), and we also assume this to be a Gaussian distribution.

![[Screenshot 2024-08-11 at 19.31.17.png]]

Here, \(\phi\) and \(\theta\) are the parameters that we need to optimize.

Proxy distributions are ditributions that are approximate of other true distributions.

Evidence Lower Bound (ELBO)

Evidence Lower Bound (ELBO)

\[\text{ELBO}(\mathbf{x})= \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right]\]

ELBO is a lower bound for the prior distribution \(\log p(\mathbf{x})\) because we can show that \(\log p(\mathbf{x}) \geq \text{ELBO}(\mathbf{x})\). There are two derivation that can lead us to the same ELBO term using Jensen’s inequality and the KL Divergence.

ELBO Derivation using Jensen’s inequality

Starting with \(\log p(\mathbf{x})\), we have: \[\begin{aligned} \log p(\mathbf{x}) &= \log \int_{\mathbf{z}} p(\mathbf{x},\mathbf{z})d\mathbf{z} \\ &= \log\int_\mathbf{z} p(\mathbf{x},\mathbf{z}) \dfrac{q_{\phi}(\mathbf{z} \vert \mathbf{x})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})} d\mathbf{z} \\ &= \log \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\dfrac{p(\mathbf{x}, \mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] \\ & \geq \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\log\dfrac{p(\mathbf{x}, \mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] && \text{(by Jensen's inequality)} \end{aligned} \]

While this derivation is straightforward, it does not give us the insight into what exactly is the difference between the ELBO term and the actual \(\log p(\mathbf{x})\). The following derivation using KL Divergence tells us explicitly that that difference is exactly the KL Divergence term between the proxy distribution of the decoder \(q_{\phi}(\mathbf{z} \vert \mathbf{x})\) and \(p(\mathbf{z} \vert \mathbf{x})\). Figure 3 gives an illustration for this gap between the ELBO and \(\log p(\mathbf{x})\).

ELBO Derivation using KL Divergence

We’ll again start with \(\log p(\mathbf{x})\) because that’s the main term that we need to find. \[\begin{aligned} \log p(\mathbf{x}) &= \log p(\mathbf{x})\int q_{\phi}(\mathbf{x} \vert \mathbf{z})d\mathbf{z} && \text{(multiply by 1)}\\ &= \int \log p(\mathbf{x})q_{\phi}(\mathbf{x} \vert \mathbf{z})d\mathbf{z}\\ &= \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})} [\log p(\mathbf{x})] && \text{(by definition of expectation)}\\ &= \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})} \left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{p(\mathbf{x} \vert \mathbf{z})}\right] && \text{(by Bayes' theorem)}\\ &= \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})} \left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{p(\mathbf{x} \vert \mathbf{z})} \times \dfrac{q_{\phi}(\mathbf{z} \vert \mathbf{x})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] && \text{(multiply and divide by 1)}\\ &= \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})} \left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})} \right] + \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})} \left[\log \dfrac{q_{\phi}(\mathbf{z} \vert \mathbf{x})}{p(\mathbf{x} \vert \mathbf{z})} \right] && \text{(split the expectation)}\\ &= \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})}\left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})} \right] + \underbrace{D_{KL}(q_{\phi}(\mathbf{z} \vert \mathbf{x}) \| p(\mathbf{x} \vert \mathbf{z}))}_{\geq 0} && \text{(by definition of KL Divergence)}\\ & \geq \mathbb{E}_{q_{\phi}(\mathbf{x} \vert \mathbf{z})}\left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})} \right] \end{aligned}\]

![[Screenshot 2024-08-13 at 10.05.37.png]]

While the ELBO is useful, still, we can not use it because of the term \(p(\mathbf{x}, \mathbf{z})\), which we don’t know. Therefore we need to do further derivation so that the ELBO only contain the terms that we know and can work with. Consider the following derivation: \[ \begin{aligned} \text{ELBO}(\mathbf{x}) &= \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\log \dfrac{p(\mathbf{x},\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] \\ &= \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\log \dfrac{p(\mathbf{x} \vert \mathbf{z})p(\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] \\ &= \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})} [\log p(\mathbf{x} \vert \mathbf{z})] + \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\left[\log \dfrac{p(\mathbf{z})}{q_{\phi}(\mathbf{z} \vert \mathbf{x})}\right] \\ &= \mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})} [\log p_{\theta}(\mathbf{x} \vert \mathbf{z})] - D_{KL}(q_{\phi}(\mathbf{z} \vert \mathbf{x}) \|p(\mathbf{z} )) \end{aligned} \] Now the ELBO only contains terms that have Gaussian distribution. In particular, we obtain that: \[\text{ELBO}(\mathbf{x}) = \underbrace{\mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})} [\log p_{\theta}(\mathbf{x} \vert \mathbf{z})]}_{\text{how good your decoder is}} - \underbrace{D_{KL}(q_{\phi}(\mathbf{z} \vert \mathbf{x}) \|p(\mathbf{z} ))}_\text{how good your encoder is}\] Note that there are now two terms in the equation, the reconstruction term and the prior matching term. The reconstruction term \(\mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})} [\log p_{\theta}(\mathbf{x} \vert \mathbf{z})]\) is about the decoder. We want the decoder to produce a good image \(\mathbf{x}\) given the latent vector \(\mathbf{z}\). Therefore, we want to maximize \(\log p_{\theta}(\mathbf{x} \vert \mathbf{z})\). The prior matching term is the KL divergence for the encoder. We want the encoder to turn an image \(\mathbf{x}\) into a latent vector \(\mathbf{z}\) such that the latent vector will be normally distributed, i.e \(\mathbf{z} \sim \mathcal{N}(0, \mathbf{I})\),

Training VAE

Encoder

To train the VAE, we need to know the ground truth pair \((\mathbf{x},\mathbf{z})\). We know that \(\mathbf{z}\) is generated from the distribution \(q_{\phi}(\mathbf{z} \vert \mathbf{x})\), and we also know that \(q_{\phi}(\mathbf{z} \vert \mathbf{x})\) is also a Gaussian. Assume that this Gaussian has mean \(\boldsymbol{\mu}\) and variance \(\sigma^{2}\mathbf{I}\). Now, the difficult part is how to obtain \(\boldsymbol{\mu}\) and covariance \(\sigma^2\mathbf{I}\) from the input image \(\mathbf{x}\). The answer is, we will construct a neural network such that \[\boldsymbol{\mu} = \underbrace{\boldsymbol{\mu}_\phi(\mathbf{x})}_{\text{neural network}}\] \[\sigma^{2}= \underbrace{\sigma^{2}_{\phi}(\mathbf{x})}_{\text{neural network}}\] Now, the \(\ell^{th}\) training sample of the training set \(\mathbf{z}^{(\ell)}\) can be sampled from the Gaussian distribution: \[\mathbf{z}^{(\ell)} \sim q_{\phi}(\mathbf{z} \vert \mathbf{x}^{(\ell)}),\quad \text{where } q_{\phi}(\mathbf{z} \vert \mathbf{x}^{(\ell)})=\mathcal{N}(\mathbf{z} \vert \boldsymbol{\mu}_\phi(\mathbf{x}^{(\ell)}), \sigma^{2}_{\phi}(\mathbf{x}^{(\ell)}))\] Here, note that \(\boldsymbol{\mu}_\phi(\mathbf{x}^{(\ell)})\) and \(\sigma^{2}_{\phi}(\mathbf{x}^{(\ell)})\) are functions of \(\mathbf{x}^{(\ell)}\), so for every different \(\mathbf{x}^{(\ell)}\), we need to come up with a different Gaussian.

Decoder

The job of the decoder network is to reconstruct the image \(\mathbf{x}\) given the latent vector \(\mathbf{z}\), which is encoded from the original image via the encoder. \[\hat{\mathbf{x}} = \text{decode}_\theta(\mathbf{z})\] One assumption that we’ll make is that the error between the reconstructed image \(\hat{\mathbf{x}}\) and the original input image \(\mathbf{x}\) is Gaussian, i.e. \[(\hat{\mathbf{x}}-\mathbf{x}) \sim \mathcal{N}(0, \sigma^{2}_\text{dec})\] It then follows that the distribution \(p_{\theta}(\mathbf{x} \vert \mathbf{z})\) is: \[ \begin{aligned} \log p_{\theta}(\mathbf{x} \vert \mathbf{z}) &= \log \mathcal{N}(\mathbf{x} \vert \text{decode}_{\theta}(\mathbf{x}), \sigma^{2}_\text{dec}\mathbf{I}) \\ &= \log \dfrac{1}{\sqrt{(2\pi \sigma^{2}_\text{dec})^{D}}} \exp \left(- \dfrac{\|\mathbf{x} - \text{decode}_{\theta}(\mathbf{z})\|^2}{2 \sigma^{2}_\text{dec}}\right)\\ &= - \dfrac{\|\mathbf{x} - \text{decode}_{\theta}(\mathbf{z})\|^2}{2 \sigma^{2}_\text{dec}}-\log\sqrt{(2\pi\sigma^2_\text{dec})^D}\end{aligned} \] where \(D\) is the dimension of \(\mathbf{x}\). Through this derivation, we know that maximizing the likelihood term in ELBO is equivalent to minimizing the \(\ell_{2}\) loss between the ground truth image and the reconstructed image, which makes a lot of sense.

Loss function

The loss function can be approximated via Monte-Carlo simulation \[\mathbb{E}_{q_{\phi}(\mathbf{z} \vert \mathbf{x})}[\log p_{\theta}(\mathbf{z} \vert \mathbf{x})] \approx \dfrac{1}{L} \sum\limits_{\ell=1}^{L}\log p_{\theta}(\mathbf{x}^{(\ell)} \vert \mathbf{z}^{(\ell)}), \quad \mathbf{z}^{(\ell)} \sim q_{\phi}(\mathbf{z} \vert \mathbf{x}^{(\ell)})\] Therefore we obtain the loss function of VAE, which can be defined as follows:

Training loss of VAE

\[\underset{\phi, \theta}{\text{argmax}} \left[\dfrac{1}{L}\sum\limits_{\ell=1}^{L}\log p_{\theta}(\mathbf{x}^{(\ell)} \vert \mathbf{z}^{(\ell)}) -D_{KL}(q_{\phi}(\mathbf{z} \vert \mathbf{x}^{(\ell)})\| p(\mathbf{z}))\right], \]

Inference

For inference, we can simply feed a latent vector \(\mathbf{z}\) which we sampled from \(p(\mathbf{z}) = \mathcal{N}(0, \mathbf{I})\) into the decoder \(\text{decode}_\theta\) and get an image \(\mathbf{x}\).

Appendix A: Proof of KL Nonnegativity

\(D_{KL}(p \| q) \geq 0\) with equality iff \(p=q\).

\[\begin{aligned} D_{\mathrm{KL}}(P\|Q)& =\sum_{x\in\mathcal{X}}P(x)\log\frac{P(x)}{Q(x)} && \text{(by definition for discrete case)} \\ &=-\sum_{x\in\mathcal{X}}P(x)\log\frac{Q(x)}{P(x)} && \text{(changing the sign of the $\log$)}\\ &=\mathbb{E}\bigg[-\log\frac{Q(x)}{P(x)}\bigg] && \text{(by Law of Unconscious stastistician)}\\ &\geq-\log \mathbb{E}\Big[\frac{Q(x)}{P(x)}\Big] && \text{(by Jensen's inequality)}\\ &=-\log\Big(\sum_{x\in\mathcal{X}}P(x)\frac{Q(x)}{P(x)}\Big) && \text{(deriving the expectation term)}\\ &=-\log\sum_{x\in\mathcal{X}}Q(x) && \text{(cancelling out the $P(x)$)}\\ &=-\log1 =0 \end{aligned}\]
